# References

This folder is the approved Lightform reference library.

It is not a screenshot hoard and it is not a clone archive. A site belongs here only if it teaches a reusable lesson for future Lightform builds.

## V1 Size Limit

Start with:

- 28 approved references
- 7 anti-pattern references

Do not scale beyond that until these entries noticeably improve first-screen concepts, strategy briefs, and build prompts.

## Entry Rules

Every approved reference must include:

- `reference.json`
- `taste-notes.md`
- `section-map.md`
- screenshots for desktop, tablet, and phone
- at least one reusable pattern card
- at least one "do not copy" note

## Approval Gate

Approve only if:

- taste score is 4 or higher
- conversion score is 4 or higher
- vertical fit score is 4 or higher
- mobile homepage was reviewed
- one key subpage was reviewed when available
- useful pattern can be explained in plain language
- no miracle claims, fake proof, or dark-pattern UX are being treated as inspiration

Reject if:

- it is only pretty
- it is generic beige wellness
- the CTA is hidden
- mobile behavior is weak
- it depends on unsupported health claims
- it cannot improve a future build prompt

## Retrieval Rule

Future site builds should retrieve only 3 to 6 entries:

- 1 primary vertical match
- 1 conversion architecture match
- 1 visual/taste match
- 1 trust/proof match
- optional contrast reference
- optional anti-pattern

Use reference lessons as design intelligence, not assets to copy.
