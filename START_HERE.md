# Start Here

This repo should stay small.

Use it to build a reference engine that future Lightform site builds can pull from.

## First Goal

Create the first 35 entries:

- 28 approved references
- 7 anti-pattern references

Then stop and test whether the library improves real site concepts. Do not keep collecting until retrieval proves useful.

## Use This Prompt

```text
Use this repo as the Lightform Design Reference Lab.

Capture this site as a reference, not as something to copy:
<url>

Create a folder under docs/references/<slug>/.

Capture screenshots, tokens, section flow, taste notes, and reusable pattern cards. Score it honestly. Do not build a clone.
```

## What Matters

Every entry should help answer:

- What client type does this reference fit?
- What customer state does it understand?
- What pattern should future builds borrow?
- What should never be copied?
- Would this improve the next generated site?

## What Does Not Matter

- scraping every page
- collecting hundreds of sites
- copying exact layouts
- saving proprietary assets for reuse
- admiring sites that do not help conversion

## Future Build Rule

Before generating a client site, retrieve only 3 to 6 references:

- one vertical match
- one conversion architecture match
- one visual/taste match
- one trust/proof match
- optional contrast reference
- optional anti-pattern

Feed only distilled lessons into the build prompt.
